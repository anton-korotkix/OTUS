# ML-прогнозирование длительности реализации RFC (Request for Change)

## Описание
В рамках текущего проекта проводится исследование прогнозирования длительности реализации продуктового запроса на изменение (Request for Change) с использованием методов машинного обучения, сравнение результатов отдельных моделей и ансамбля из них.

## Цель
Сравнить несколько моделей, которые по характеристикам RFC спрогнозируют предполагаемую длительность реализации в днях.
Данная задача актуальна для ИТ-компаний с продуктовым конвейером разработки, т.к. позволит оптимизировать планирование спринтов и релизов бизнес-задач.

## Датасет
- Id - идентификатор RFC
- Priority - бизнесовый приоритет (1-3)
- Systems - количество задействованных систем
- Integrations - количество межсистемных интеграций
- Enterprise architecture - статус готовности Enterprise-архитектуры (in progress/done/new)
- Technical debt - наличие технического долга (True/False)
- PM - менеджер проекта
- Days - количество дней на реализацию
- Effort - трудозатраты

## Инструменты
Для исследования используются:
- RandomForestRegressor - модель под задачи регрессии
- KNeighborsRegressor - модель под задачи регрессии
- VotingRegressor - ансамблевый метод
- SimpleImputer - обработка пропусков в данных
- OneHotEncoder - обработка категориальных данных
- StandardScaler - нормализация данных

## План
- Разведочный анализ данных (EDA) и определение целевой переменной
- Визуализация распределения признаков и матрица корреляций
- Предобработка данных
- Пайплайн, моделирование и ансамбль
- Сравнение метрик
- Заключительный вывод
